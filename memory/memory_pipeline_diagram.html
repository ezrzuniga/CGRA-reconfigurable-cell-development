PE Memory Cell Pipeline

This diagram shows the conceptual data/control flow of the memory cell as a pipeline for DMA-style transfers.

1. Host / CSR

Programs context registers and issues start commands.

src_addr, dst_addr, stride, count, mode, dir
→

2. PE Memory Cell

Receives CSR transactions and dispatches the DMA job.

Context selection + control register handling
→

3. DMA Engine

Uses address controllers to generate the transfer sequence.

Direct or strided accesses
→

4. SRAM Bank

Reads source words or writes destination words.

Dual-port memory backend
→

5. NoC / Peer Interface

Transfers data to the network or another memory tile.

SRAM ↔ NoC / SRAM ↔ SRAM
→

6. Status / Done

Busy/done flags are reported back to the host.

CSR status register
Control path: CSR Data path: SRAM ↔ NoC Execution path: DMA engine
